# Nimble Tiger Plan — Exponential, Self-Improving Coding Agents

## Goals
- Build a safe, scalable framework that spawns specialized coding agents to tackle tasks, with continual improvement of agents and artifacts.
- Maintain governance, safety, and reproducibility while enabling rapid exploration of ideas.

## Assumptions
- Access to a local repository with a task planner, agent templates, and a sandboxed execution environment.
- We will not implement self-destructive behavior or bypass safeguards.
- Growth is gradual and controllable via quotas and review gates.

## System Architecture
- Orchestrator: coordinates tasks, decomposes work, assigns agents, collects feedback.
- AgentTemplates: blueprints for domains (frontend, backend, tests, docs, tooling).
- AgentInstances: spawned from templates with task-specific prompts and evaluation criteria.
- KnowledgeBase: stores outputs, experiments, templates, and learned improvements.
- EvaluationEngine: automated checks (lint, tests, type checks) and qualitative review.
- SafetyGate: rate limits, sandboxing, and human-in-the-loop when needed.

## Agent Model
- Each agent has:
  - Objective, constraints, success criteria
  - Domain capabilities and tool access
  - Self-improvement plan: after task, propose template refinements and new templates
  - Output artifacts: code, diffs, docs, tests, PRs

## Orchestration & Planning
- Planner decomposes tasks into subtasks; assigns to best-fit agent.
- Spawn policy uses diversification: rotate templates, parameterize prompts, and explore alternative solutions.
- Versioning for agent templates and outputs; rollbacks as needed.
- Global loop: evaluation results update templates and knowledge base.

## Growth Strategy
- Exponential growth via combinatorial task decomposition and template diversification.
- Capped growth with quotas and human checks; prune underperforming agents.
- Meta-learning: track performance metrics and refine templates.

## Evaluation & Safety
- Metrics: correctness, completeness, maintainability, speed, adherence to specs.
- Automated checks: lint, typecheck, tests; integration checks for interfaces.
- Safety: sandboxed execution, input/output sanitization, no access to secrets.
- Human review gate before deploying new agent generations beyond a small initial set.

## Roadmap & Milestones
- M0: Baseline orchestrator + 3 domain templates; simple feedback loop.
- M1: Automated evaluation pipeline; template refinement triggers.
- M2: First wave of new-agent templates; growth controls in place.
- M3: Meta-learning loop; continuous improvement with governance.

## Risks & Mitigations
- Risk: uncontrolled growth. Mitigation: quotas, audits, and human-in-the-loop.
- Risk: quality drift. Mitigation: strong evaluation criteria and rollback.
- Risk: security leaks. Mitigation: sandboxing and secret management policies.
